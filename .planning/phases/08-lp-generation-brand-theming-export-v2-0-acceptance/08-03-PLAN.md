<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Serve origin → index.html: workspaceId from HMAC | workspaceId para lookup do BrandConfig vem das claims HMAC verificadas — não do URL ou de cookie de sessão (serve origin não tem sessão PageForge) |
| LP preview RSC → iframe URL: entryPath | entryPath vem de `lp.entryRoute` (lido do banco, workspace-scoped via TenantLpHelpers) + default `/`; nunca de query string não validada |
| iframe sandbox | sandbox="allow-scripts" sem allow-same-origin mantém origem opaca; document.cookie no iframe não expõe cookies do dashboard |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-03-01 | Tampering | BrandConfig.primaryColor → CSS injection | accept | primaryColor validado em brand/schema.ts como `/^#[0-9a-fA-F]{6}$/`; triplet HSL contém apenas dígitos, `%` e espaços — sem vetor de CSS injection. buildBrandStyleTag usa template string literal com valor pré-validado |
| T-08-03-02 | Information Disclosure | serve route: BrandConfig lookup via bare prisma | mitigate | workspaceId vem exclusivamente das claims HMAC verificadas (assinadas com SERVE_TOKEN_SECRET, expiradas em 30min); bare `prisma.brandConfig.findFirst({ where: { workspaceId } })` não requer sessão mas é scoped ao workspaceId trusted do token |
| T-08-03-03 | Information Disclosure | iframe sandbox breakout | mitigate | sandbox="allow-scripts" sem allow-same-origin mantém origem opaca; mesmo que o JS do SPA tente `parent.postMessage`, o dashboard só aceita mensagens de origens conhecidas (pattern herdado da Fase 7) |
| T-08-03-04 | Elevation of Privilege | entryPath no iframe URL | accept | entryPath é `lp.entryRoute ?? "/"` — string do banco, não de URL param não validado; o serve handler usa SPA fallback para qualquer path → não expõe arquivo inesperado |
| T-08-03-05 | Tampering | transformToString() vs transformToWebStream() — stream consumido duas vezes | mitigate | O body S3 é consumido exatamente uma vez: index.html via transformToString(), assets via transformToWebStream(). A lógica de branch garante execução mutuamente exclusiva por request |
</threat_model>
