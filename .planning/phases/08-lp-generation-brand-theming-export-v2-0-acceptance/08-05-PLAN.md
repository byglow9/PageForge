  <what-built>
    Os Planos 01–04 implementaram o ciclo de vida completo de LP VITE_SPA:
    - Migração `entry_route` aplicada ao banco
    - `lib/brand/theme.ts` com hexToHslTriplet + injeção de CSS vars
    - `generateViteSpaLpAction` com branch VITE_SPA em generate/update/duplicate
    - `ViteSpaLpForm` na página de geração
    - Serve handler com brand injection no index.html
    - Preview page com iframe sandboxed + entryRoute na URL
    - Export route VITE_SPA: ZIP da árvore dist/ + index.html tematizado + sem CSP
    - Edit page com branch VITE_SPA → ViteSpaLpForm pré-preenchido
  </what-built>
  <how-to-verify>
    Execute o roteiro de UAT v2.0 abaixo. Anote PASS/FAIL para cada item.

    **Pré-condição:** servidor de dev rodando (`pnpm --filter web dev`), BrandConfig com `primaryColor` configurado (ex: `#0d4080`), template renova-turismo VITE_SPA cadastrado (Task 0).

    ---

    **Bloco A — Geração por rota**

    A1. Navegar para `/w/{slug}/lps/new` → selecionar o template `renova-turismo` (VITE_SPA).
    → Esperado: página mostra "Generate Landing Page" com campo "Landing page name" e campo "Entry route (optional)".

    A2. Preencher `name = "Renova Turismo — Grécia"`, `entryRoute = "/grecia"`, clicar "Generate landing page".
    → Esperado: sucesso toast "Landing page created." + redirecionamento para preview.

    A3. Verificar no banco: `SELECT name, kind, entry_route FROM landing_page ORDER BY created_at DESC LIMIT 1;`
    → Esperado: `kind = 'VITE_SPA'`, `entry_route = '/grecia'`.

    ---

    **Bloco B — Preview com brand theming**

    B1. Na página de preview (deve ter redirecionado automaticamente após A2): o iframe carrega o SPA.
    → Esperado: iframe visível com o SPA do renova-turismo renderizado na rota Grécia.

    B2. Abrir DevTools → inspecionar o HTML retornado pela origem isolada (ex: `http://{tplId}.serve.localhost:3000/grecia`).
    → Esperado: `<style>:root{--primary:213 90% 23%;}</style>` (ou o HSL da cor configurada no BrandConfig) antes de `</head>`.

    B3. Verificar no browser que a cor primária da página está aplicada (botões, links em `--primary`).
    → Esperado: elementos com `hsl(var(--primary))` na cor configurada no workspace.

    ---

    **Bloco C — Export ZIP tematizado**

    C1. Na LP gerada, clicar em "Export ZIP" no menu kebab.
    → Esperado: download de arquivo `.zip` com o nome slugificado da LP.

    C2. Descompactar o ZIP e abrir `index.html` em editor de texto.
    → Esperado: contém `<style>:root{--primary:...;}</style>` antes de `</head>`.
    → Esperado: NÃO contém `<meta http-equiv="Content-Security-Policy" content="script-src 'none'">`.
    → Esperado: pasta `assets/` presente com arquivos JS/CSS do bundle.

    C3. Abrir o `index.html` do ZIP diretamente no browser (arquivo local).
    → Esperado: o SPA carrega e exibe a rota Grécia (ou a raiz, dependendo do client-side router do bundle).

    ---

    **Bloco D — Editar e duplicar**

    D1. Navegar para `/w/{slug}/lps/{lpId}/edit`.
    → Esperado: "Edit Landing Page" com `name` e `entryRoute = /grecia` pré-preenchidos.

    D2. Alterar `entryRoute` para `/turquia`, clicar "Save changes".
    → Esperado: toast "Landing page updated." e `entry_route = '/turquia'` no banco.

    D3. Duplicar a LP via menu kebab → "Duplicate".
    → Esperado: toast "Duplicate created." e nova LP com `kind=VITE_SPA`, `entry_route=/turquia`, `name='Copy of Renova Turismo — Grécia'`.

    ---

    **Bloco E — Coexistência v1 (Grécia LIQUID intacto)**

    E1. Navegar para o template Liquid Grécia e gerar uma LP LIQUID (ou abrir uma existente).
    → Esperado: formulário dinâmico com todos os campos (não ViteSpaLpForm).

    E2. Preview de LP LIQUID → mostra HTML renderizado (não iframe).
    → Esperado: HTML da Grécia visível no browser sem iframe.

    E3. Export de LP LIQUID → ZIP com `index.html` contendo o HTML LIQUID com CSP.
    → Esperado: o ZIP contém `script-src 'none'` no `index.html` (CSP do LIQUID preservada).

    E4. Catálogo `/w/{slug}/lps` — ambas as LPs visíveis; LP VITE_SPA com badge "Vite SPA", LP LIQUID com badge "LIQUID" (ou sem badge — verificar o que a Fase 5 implementou).

    ---

    **Resultado esperado:** Todos A1–E4 passam.
