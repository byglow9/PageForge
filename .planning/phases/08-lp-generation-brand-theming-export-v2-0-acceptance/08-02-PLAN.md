## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Cliente → generateViteSpaLpAction | Input chega via Server Action; templateId e entryRoute são untrusted client input |
| generateViteSpaLpAction → banco | workspaceId derivado exclusivamente da sessão server-side via requireWorkspaceRole |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-02-01 | Spoofing | generateViteSpaLpAction: workspaceId | mitigate | `requireWorkspaceRole` deriva workspaceId da sessão; nunca aceita workspaceId do payload do cliente |
| T-08-02-02 | Tampering | entryRoute: path injection | mitigate | `GenerateViteSpaLpSchema.entryRoute` valida `.max(128)` e normaliza para null; o path é armazenado e usado como URL path em serve origin — não executado como código; o serve handler trata qualquer path extensionless como SPA fallback para index.html |
| T-08-02-03 | Elevation of Privilege | VITE_SPA template de outro workspace | mitigate | `db.template.findById` usa `TenantTemplateHelpers` com workspaceId scoping (RLS); não é possível gerar LP de template de outro workspace |
| T-08-02-04 | Repudiation | duplicateLpAction VITE_SPA sem assets | accept | Duplicação VITE_SPA é uma criação de nova linha com referência ao mesmo template; auditoria via timestamps normais do Prisma — sem impacto de segurança adicional |
