## Spacing Scale

Declared values (must be multiples of 4). Matches the existing dashboard shell conventions (px-8 py-6 page wrapper, gap-2 footer, gap-1 badge rows).

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge gap within chip rows |
| sm | 8px | Compact element spacing, button inner gap |
| md | 16px | Default element spacing, form field stack gap |
| lg | 24px | Section padding (page `py-6`), card content gap |
| xl | 32px | Layout gaps, page horizontal padding (`px-8`) |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Preview iframe: `height: 80vh` (viewport-relative, not spacing-token-based) — mirrors existing project-template preview pattern.
- Touch targets for kebab/icon buttons: minimum 40px hit area via `p-2` padding on a `h-4 w-4` icon.

**Source:** Existing `LpCatalogCard` (`gap-1`, `gap-2`, `mt-2`), page shells (`px-8 py-6`), project-template preview page (`style={{ width: "100%", height: "80vh" }}`).

## Typography

Declared type scale — exactly 4 sizes:

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (`text-sm`) | 400 regular | 1.5 |
| Label | 14px (`text-sm`) | 400 regular | 1.4 |
| Heading | 24px (`text-2xl`) | 600 semibold | 1.2 |
| Display | 20px (`text-xl`) | 600 semibold | 1.2 |

Notes:
- Two weights only: 400 (regular / `font-normal`) and 600 (semibold / `font-semibold`). Weight 500 (medium / `font-medium`) is not used in this phase.
- Body and Label share the same size and weight. Visual distinction between them is achieved via color: Body uses `text-gray-900`; Label uses `text-gray-500` for secondary/hint text.
- Page title uses `text-2xl font-semibold text-gray-900` — existing pattern from `NewLpFromTemplatePage` (`"Generate Landing Page"` heading) and LP edit page.
- Card title uses `text-base font-semibold text-gray-900` — existing `LpCatalogCard` `CardTitle` pattern.
- Metadata text (dates, schema version) uses `text-sm text-gray-500` or `text-sm text-gray-400`.
- Font family: Space Grotesk (`--font-display`) from `globals.css @theme`.
- Badge text size is a shadcn component-internal default (not a declared scale size) and is excluded from the type scale. Do not add an explicit `text-xs` class to `<Badge>` elements — let the component apply its own internal sizing.

**Pre-populated from:** existing page shells and `LpCatalogCard`. Weight collapse to 2 weights applied per checker dimension 4 rule.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#f8fafc` (oklch 1 0 0 background) | Page background, card backgrounds (white), grid background |
| Secondary (30%) | `oklch(0.97 0 0)` — near-white gray | Secondary elements, muted surfaces, input backgrounds, sidebar |
| Accent (10%) | `oklch(0.205 0 0)` — near-black (dashboard primary) | Primary CTA buttons, active states, focus rings |
| Destructive | `oklch(0.577 0.245 27.325)` — red | Destructive actions only (delete LP confirmation button) |

Accent reserved for:
- "Generate landing page" submit button (primary CTA on the VITE_SPA generation form)
- "Save" / "Update route" submit button on the VITE_SPA LP edit form
- Focus ring on the entry route input field when active
- No other elements

Additional color notes:
- `text-red-600 focus:text-red-600 focus:bg-red-50` — "Delete landing page" DropdownMenuItem (existing pattern from `LpCatalogCard`)
- `text-gray-400` / `text-gray-500` — metadata, secondary labels, placeholder text, hint text under fields (distinguishes Label role from Body without a third weight)
- `text-gray-700` — body-level interactive labels (Preview/Edit inline links)
- Badge variants: `secondary` (gray) for folder and tag chips; `outline` for `VITE_SPA` kind discriminator badge (existing `LpCatalogCard` pattern)

**Pre-populated from:** `globals.css @theme` + `LpCatalogCard` patterns.

### New component: `ViteSpaLpForm`

Location: `apps/web/src/components/lps/ViteSpaLpForm.tsx`

A minimal client component for VITE_SPA LP generation and editing. It shares the same page-wrapper pattern as `LpForm` but is structurally simpler: only two fields (name + optional entry route), plus a submit button.

**Primary focal point (generation mode):** The "Landing page name" input receives `autoFocus` on mount, establishing it as the entry point. The "Generate landing page" CTA is the visual anchor at the bottom of the form — it is the only accent-colored element on the page. Tab order: name → entryRoute → submit.

**Generation mode layout:**
```
<div class="px-8 py-6">
  <h1 class="text-2xl font-semibold text-gray-900 mb-6">Generate Landing Page</h1>

  <form class="space-y-6 max-w-lg">
    <!-- LP Name (autofocused — primary focal point) -->
    <div class="space-y-2">
      <Label for="name">Landing page name</Label>
      <Input id="name" placeholder="e.g. Grécia — Outubro 2026" required autoFocus />
    </div>

    <!-- Entry Route (optional) -->
    <div class="space-y-2">
      <Label for="entryRoute">Entry route <span class="text-gray-400 font-normal">(optional)</span></Label>
      <Input id="entryRoute" placeholder="e.g. /grecia" />
      <p class="text-sm text-gray-500">
        Leave blank for single-page projects (defaults to /). For multi-route
        projects, type the path you want to link to (e.g. /grecia).
      </p>
    </div>

    <!-- Submit (visual anchor — only accent element on page) -->
    <Button type="submit" disabled={isPending}>
      {isPending ? "Generating…" : "Generate landing page"}
    </Button>
  </form>
</div>
```

**Edit mode layout:**
```
<div class="px-8 py-6">
  <h1 class="text-2xl font-semibold text-gray-900 mb-6">Edit Landing Page</h1>

  <form class="space-y-6 max-w-lg">
    <!-- LP Name -->
    <div class="space-y-2">
      <Label for="name">Landing page name</Label>
      <Input id="name" defaultValue={lp.name} required />
    </div>

    <!-- Entry Route -->
    <div class="space-y-2">
      <Label for="entryRoute">Entry route <span class="text-gray-400 font-normal">(optional)</span></Label>
      <Input id="entryRoute" defaultValue={lp.entryRoute ?? ""} placeholder="e.g. /grecia" />
      <p class="text-sm text-gray-500">
        Leave blank for the root page (/). The preview will reload with the
        updated route after saving.
      </p>
    </div>

    <!-- Submit -->
    <Button type="submit" disabled={isPending}>
      {isPending ? "Saving…" : "Save changes"}
    </Button>
  </form>
</div>
```

### LP Preview page — VITE_SPA branch

Mirrors the existing `project-templates/[id]/preview/page.tsx` layout exactly. Replace `renderLp()` call with token-mint + iframe pattern.

```
<div class="px-8 py-6 space-y-4">
  <div class="flex items-center justify-between">
    <h1 class="text-2xl font-semibold text-gray-900">{lp.name}</h1>
    <Badge variant="outline">Vite SPA</Badge>
  </div>

  <!-- Entry route label (informational) -->
  {lp.entryRoute && (
    <p class="text-sm text-gray-500">Route: {lp.entryRoute}</p>
  )}

  <!-- Cross-origin sandboxed iframe.
       sandbox="allow-scripts allow-same-origin" (T-08-03-03 revisado): allow-same-origin
       é necessário porque o entry do Vite é <script type="module" crossorigin> — sob
       origem opaca o módulo é bloqueado por CORS e localStorage lança SecurityError
       (preview em branco). Isolamento vem do subdomínio de serve cross-origin +
       cookies host-only + CSP frame-ancestors, NÃO da origem opaca. -->
  <iframe
    src={`${serveOrigin}${entryPath}?t=${token}`}
    sandbox="allow-scripts allow-same-origin"
    style={{ width: "100%", height: "80vh", border: "none" }}
    title={`Preview: ${lp.name}`}
  />

  <!-- Back link -->
  <div class="flex justify-end">
    <a href={`/w/${slug}/lps`} class="...existing button-link style...">
      Back to catalog
    </a>
  </div>
</div>
```

### Accessibility: kebab menu icon-only triggers

`DropdownMenuTrigger` elements that contain only an icon (no visible text) must include `aria-label="LP actions"`. This is an inherited pattern from `LpCatalogCard` — all VITE_SPA LP card kebab triggers follow the same rule. Example:

```tsx
<DropdownMenuTrigger asChild>
  <Button variant="ghost" size="icon" aria-label="LP actions">
    <MoreHorizontal className="h-4 w-4" />
  </Button>
</DropdownMenuTrigger>
```

## States and Interactions

### VITE_SPA LP Generation Form

| State | Visual |
|-------|--------|
| Idle | Name input autofocused (primary focal point), entry route empty, "Generate landing page" button enabled |
| Submitting | Button shows "Generating…" with `disabled`, spinner via `Loader2` icon if needed |
| Success | Toast: "Landing page created." + redirect to `/w/{slug}/lps/{id}/preview` |
| Error (action fails) | Toast: "Failed to generate. Try again." Button re-enables |
| Validation: name empty | Input border highlights, no inline error text (native browser required validation) |

### VITE_SPA LP Edit Form

| State | Visual |
|-------|--------|
| Idle | Name and entry route pre-populated from DB, "Save changes" button enabled |
| Submitting | Button shows "Saving…" with `disabled` |
| Success | Toast: "Landing page updated." + `router.refresh()` |
| Error | Toast: "Failed to save. Try again." |

### LP Preview — VITE_SPA

| State | Visual |
|-------|--------|
| Loading | Iframe renders blank until serve origin responds (no explicit loading state in RSC shell; browser handles iframe load) |
| Loaded | Iframe displays the SPA with brand `--primary` injected |
| Error (serve origin unreachable) | Full-page fallback: "Preview failed to load. Try refreshing." + "Refresh preview" button + Back to catalog link (same pattern as `project-templates/[id]/preview`) |

### Export ZIP — VITE_SPA

| State | Visual |
|-------|--------|
| Triggered | `isExporting = true` → DropdownMenuItem shows spinner + "Exporting…" text (same as existing LIQUID export in `LpCatalogCard`) |
| Success | Toast: "Export ready — downloading." + browser triggers file download |
| Error | Toast: "Export failed. Try again." `isExporting` resets |

### LP Catalog Card — VITE_SPA LP

- Shows `Badge variant="outline"` with text `Vite SPA` below the card title (existing pattern from `LpCatalogCard`).
- All other card interactions (Preview link, Edit link, Export ZIP, Move to folder, Edit tags, Duplicate, Delete) are unchanged — VITE_SPA LPs flow through identically.
- `schemaVersion` badge (v0) is shown in the card header (schema version exists as sentinel 0 for VITE_SPA; executor may choose to hide the version badge for VITE_SPA rows — acceptable discretion).

## Design Decisions Inherited from Upstream

| Decision | Source | Impact on UI |
|----------|--------|-------------|
| Entry route defaults to `/` (D-01) | CONTEXT.md | Field is optional; placeholder and hint communicate the default clearly without forcing input |
| No route parsing from bundle (D-02) | CONTEXT.md | No dropdown picker for routes; free-text Input only |
| Brand theme is live, not snapshot (D-04) | CONTEXT.md | No UI needed to "apply theme" — preview always reflects current brand. No theme-snapshot timestamp shown |
| MVP injects only `--primary` (D-05) | CONTEXT.md | No UI for configuring which CSS vars to inject. Brand Settings page (existing) is the only control |
| VITE_SPA LP references template (not copies dist/) (D-07) | CONTEXT.md | No storage cost indicator in UI. No "LP will break if template is deleted" warning in v2.0 MVP |
| Reuse `LandingPage` table (D-08) | CONTEXT.md | Catalog cards, folders, tags, duplications all unchanged |
| Edit = route + name only (D-09) | CONTEXT.md | Edit form is minimal (2 fields). No rich field editor, no schema panel |
| Branch by kind in existing export route (D-10) | CONTEXT.md | Export ZIP menu item is identical for VITE_SPA and LIQUID from the user's perspective |
| No strict CSP for VITE_SPA export (D-12) | CONTEXT.md | No UI impact |
