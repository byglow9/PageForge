<objective>
Create the isolated serving layer: a `proxy.ts` that routes `*.serve.*` host requests to an internal catch-all route, and a serving route handler at `app/serve/[tplId]/[[...path]]/route.ts` that validates the HMAC token, enforces cross-tenant isolation, applies the type guard, resolves S3 keys with SPA fallback, and streams bytes with correct security headers.

This plan implements the non-retrofittable origin isolation decision (D4): after this plan, the dist/ of a VITE_SPA template is served from a host that shares no cookies with the dashboard origin.

Purpose: Delivers PRJ-04 (isolated serving origin) and PRJ-06 (cross-tenant isolation, non-enumerable keys). The MVP tradeoff (per RESEARCH.md Pitfall 1 and Open Question 1) is explicit: the HMAC token is required only for the initial index.html request; subsequent asset requests are authorized by the non-enumerable tplId UUID segment in the S3 key path.

Output:
- apps/web/src/proxy.ts — host detection (Next.js 16 proxy.ts convention)
- apps/web/src/app/serve/[tplId]/[[...path]]/route.ts — streaming serving handler
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → proxy.ts | Host header arrives from HTTP client; never trusted for authorization — proxy only rewrites path |
| Query param ?t= → verifyServeToken | Token is untrusted input from the browser; full HMAC verify before any data access |
| tplId URL segment → S3 key | tplId from URL used in S3 key construction — validated against DB (template must exist) and against token claims |
| S3 key → S3 object | Key derived server-side from token claims + DB; never derived from raw URL parameters alone for index.html requests |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-02-01 | Elevation of Privilege | Cross-tenant access — workspace A reaching workspace B's dist/ | mitigate | Token HMAC payload contains workspaceId+templateId; handler validates claims.templateId === tplId from URL; DB lookup confirms template belongs to claimed workspaceId; mismatched token → 403 |
| T-07-02-02 | Tampering | HMAC token forgery / signature bypass | mitigate | verifyServeToken uses timingSafeEqual (Plan 01 T-07-01-01); signature recomputed server-side from the same secret; any bit flip → null → 403 |
| T-07-02-03 | Tampering | Token expiry tampering — advancing exp field | mitigate | exp is inside the HMAC-signed payload; modifying exp changes the signed data → signature mismatch → 403 |
| T-07-02-04 | Information Disclosure | S3 key enumeration via path traversal | mitigate | resolveServePath strips leading slashes, never produces paths starting with '..'; S3 key prefix `workspaces/{wId}/project-templates/{tplId}/dist/` is prepended server-side; tplId is a UUID (non-enumerable) — exhaustive enumeration is infeasible |
| T-07-02-05 | Information Disclosure | Session cookie theft by third-party JS in served SPA | mitigate | Isolation enforced at the proxy level (separate host) — no PageForge session cookie is present on *.serve.* origins; serving handler must NOT set Set-Cookie for PageForge session cookies; CSP frame-ancestors prevents unauthorized iframe embedding |
| T-07-02-06 | Elevation of Privilege | Type confusion — LIQUID template reaching VITE_SPA serving path | mitigate | assertViteSpaKind(template.kind) called before any S3 access; LIQUID → 403 "Type boundary violation" (D-08); mirrors the renderLp() guard from Phase 6 Plan 01 |
| T-07-02-07 | Tampering | S3 Body stream consumed before streaming to Response | mitigate | Content-Type derived from file extension via getContentType() — NEVER read Body bytes for MIME detection (Pitfall 4 prevention); Body.transformToWebStream() called exactly once |
| T-07-02-08 | Tampering | CSP frame-ancestors in meta tag (ineffective) | mitigate | frame-ancestors set exclusively as HTTP response header in NextResponse headers object (Pitfall 6 prevention); no meta tag CSP injection in this handler |
| T-07-02-09 | Repudiation | Token replay within TTL window | accept | Scope limits impact to 1 template for 30 minutes; bucket is not public; nonce/jti one-time token deferred to post-MVP per RESEARCH.md |
| T-07-02-10 | Information Disclosure | Dashboard host detection in proxy (host spoofing) | accept | The proxy reads the Host header for routing decisions, not for authorization; authorization happens in the route handler after rewrite; spoofed Host that doesn't match SERVE_HOST_RE passes to dashboard — no data exposed |
</threat_model>
