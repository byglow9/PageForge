<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| process.env → token module | SERVE_TOKEN_SECRET must be present; absence causes runtime panic at first token operation |
| token string → verifyServeToken | Token arrives from query-string (untrusted); full parse/verify cycle is the trust boundary |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01-01 | Tampering | verifyServeToken — signature comparison | mitigate | Use timingSafeEqual from node:crypto; never use === or Buffer.equals() for sig comparison (prevents timing oracle) |
| T-07-01-02 | Repudiation | mintServeToken — exp field | mitigate | exp = Date.now() + 30min baked into HMAC payload; verifyServeToken checks exp <= Date.now() and returns null (token expiry enforced at verify time, not just issued) |
| T-07-01-03 | Elevation of Privilege | SERVE_TOKEN_SECRET absent from env | mitigate | process.env.SERVE_TOKEN_SECRET! will throw at module load if undefined (fail-fast); document required env var; add to .env.local with placeholder |
| T-07-01-04 | Information Disclosure | assertViteSpaKind error message | accept | Error message reveals the kind value received; acceptable because this is server-side only — error is logged, not returned to the client as a response body |
| T-07-01-05 | Tampering | resolveServePath — path traversal via requestPath | mitigate | Leading slash stripped via replace(/^\/+/,''); result never starts with '..'; actual key construction in the route handler (Plan 02) prepends the S3 prefix — traversal cannot escape the template's dist/ prefix |
</threat_model>
