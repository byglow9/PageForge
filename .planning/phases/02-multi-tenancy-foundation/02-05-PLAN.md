Server Action binding pattern for forms with hidden/controlled inputs:
  The .bind() pattern pre-fills slug and memberId as positional args.
  For input fields (email, role select), the Server Action must read from FormData.
  This means the action signatures need to accept FormData as last parameter
  OR the page must use a wrapper approach.

  Recommended approach (matches existing codebase style in actions.ts):
  - Wrap the form submission in an inline async server function using bind:
      createInvitationAction.bind(null, slug)
  - The action receives (slug: string, formData: FormData) — adjust the action signature
    to accept either the current CreateInvitationInput object OR a FormData variant.

  Alternatively: keep current signatures and use hidden inputs + FormData extraction in the action.
  Either approach is acceptable; choose the one that requires fewer changes to existing tests.

  Simplest safe approach: Add FormData-accepting overloads or wrap at the page level using
  a dedicated per-page "use server" inline action. The inline action approach isolates the
  FormData extraction to the page and keeps the core action signature unchanged.
