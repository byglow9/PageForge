<objective>
Fix the four critical authorization gaps in the invitation flow and extend RLS to real tenant tables.

Purpose: SC-2 fails because acceptInvitation has no email match check (any verified user can accept any invitation ID), acceptance is triggered via GET (CSRF), and the upsert silently overwrites existing member roles. SC-4 fails because RLS protects only the probe table; workspace_member, workspace_invitation, and workspace have no ENABLE/FORCE ROW LEVEL SECURITY. This plan closes both gaps at the code and migration level — the migration is written here and applied in 02-05.

Output:
- invitations.ts: email match guard + safe upsert (no role overwrite on existing member)
- invitations/[id]/page.tsx: acceptance converted from GET render to POST Server Action
- tenant-db.ts: SET LOCAL rewritten as SELECT set_config($1, $2, true) parameterized call
- 0002_rls_real_tenant_tables/migration.sql: ENABLE/FORCE RLS + current_setting policy on workspace_member, workspace_invitation, workspace
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| invitation URL → acceptInvitation | Invitation ID in URL is an opaque bearer token; any user who learns the ID can request acceptance — email match is the authorization gate |
| GET request → membership mutation | Browser prefetch, email scanners, img tags can trigger GET URLs automatically — state-changing operations must require POST |
| withTenantDb workspaceId parameter | workspaceId string flows from server session through withTenantDb into SET LOCAL; future callers may pass unvalidated values |
| migration SQL → Postgres RLS | RLS policies must be applied via migration; they do not self-heal if the migration is absent |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04-01 | Elevation of Privilege | acceptInvitation — missing email match | mitigate | Add invitation.email.toLowerCase() !== user.email.toLowerCase() check before upsert; throw on mismatch (CR-01) |
| T-02-04-02 | Spoofing | invitations/[id]/page.tsx — GET mutation (CSRF) | mitigate | Convert to POST Server Action with Next.js CSRF protection; remove all state mutation from GET render path (CR-03) |
| T-02-04-03 | Elevation of Privilege | acceptInvitation upsert overwrites existing role | mitigate | Change upsert update clause to {} (no-op); existing members are not affected by re-accept (WR-04) |
| T-02-04-04 | Tampering | tenant-db.ts $executeRawUnsafe string interpolation | mitigate | Replace with $executeRaw parameterized set_config() call — value becomes a bind parameter (WR-01) |
| T-02-04-05 | Information Disclosure | workspace_member / workspace_invitation — no RLS | mitigate | Add ENABLE/FORCE ROW LEVEL SECURITY + current_setting policy in 0002 migration; applied in 02-05 (CR-02) |
| T-02-04-06 | Elevation of Privilege | workspace table — no RLS | mitigate | Add tenant_isolation policy on workspace.id in 0002 migration (CR-02) |
</threat_model>
