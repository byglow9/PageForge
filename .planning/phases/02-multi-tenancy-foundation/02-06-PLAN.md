<objective>
Add per-endpoint / per-server-action cross-tenant integration tests that run against the live PostgreSQL database with RLS active.

Purpose: SC-3 fails because existing isolation tests mock Prisma and prove only the app-layer WHERE clause shape — they do not prove DB-level denial. The VERIFICATION.md requirement is explicit: tests must attempt to read or edit workspace B's data BY ID using workspace A's session and assert denial, exercised against real PostgreSQL with the migration applied. This plan adds those tests now that the DB is live and RLS is applied (02-05 prerequisite).

Output:
- tenant-isolation.test.ts: new "DB-required" integration suite that uses the live DB to prove RLS blocks cross-tenant reads when SET LOCAL is absent, and allows reads when correctly set
- invitations.test.ts: integration test that creates real invitation + user records and proves acceptInvitation throws on email mismatch
- Tests are conditional on DATABASE_URL presence (skip gracefully if not set, fail if set and RLS not active)
</objective>
