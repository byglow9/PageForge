# Phase 10 — UI Design Contract

> Visual and interaction contract for the in-iframe text editor (VITE_SPA preview). Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> Scope: TEXT editing only. The editor shell (toolbar, banner, selection, postMessage, save/discard) must be architected so image/link (Phase 11) and a future color control plug in without rework (D-04). This spec covers the **dashboard parent UI** (Tailwind, design-system components) AND the **in-iframe highlight CSS** (cross-origin — standalone inline-style values, NO Tailwind available).

---

## Design System

| Property | Value | Source |
|----------|-------|--------|
| Tool | shadcn (already initialized) | `apps/web/components.json` |
| Preset | `base-nova`, baseColor `neutral`, cssVariables true | `components.json` |
| Component library | Base UI (`@base-ui/react`) via shadcn wrappers | `components/ui/button.tsx`, `badge.tsx` |
| Icon library | lucide-react | `components.json` `iconLibrary: lucide` |
| Font | System sans (body); Space Grotesk via `font-display` (`--font-space-grotesk`) for headings | `globals.css` |
| Existing primitives to reuse | `Button`, `Badge`, `Dialog`, `Tooltip`, `Separator`, `Alert` | `components/ui/` |

**Cross-origin caveat (load-bearing):** The preview iframe is served from a distinct origin (`{tplId}.serve.localhost` / `serve.{SERVE_DOMAIN}`). The injected edit script CANNOT use Tailwind classes or the app's CSS variables. All in-iframe visual feedback (hover outline, selection highlight, contentEditable affordance) MUST be applied as inline styles with the literal CSS values declared in the **In-Iframe Highlight Contract** below.

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| xs | 4px | Icon-to-label gap inside buttons/badges |
| sm | 8px | Gap between toolbar buttons; banner inline padding |
| md | 16px | Toolbar horizontal padding (`px-4`); gap between toolbar groups |
| lg | 24px | Confirmation dialog padding |
| xl | 32px | — |

Phase-specific fixed dimensions:
- Edit toolbar height: **48px** (`h-12`) — matches the existing preview-page toolbar pattern.
- Active-edit banner height: **32px** — fixed via `h-8` with `py-1` (4px) vertical padding on 14px text; the fixed height preserves the 32px strip without a 6px (non-grid) padding value.
- Iframe active-edit border: **3px** solid (see Color).

Exceptions: none. All values are multiples of 4.

## Color

The dashboard is a **neutral** surface (oklch grays). Phase 10 introduces ONE reserved accent — **edit-mode blue** — used exclusively to signal "you are editing." This makes the mode unmistakable and never competes with the neutral catalog chrome.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#f8fafc` page grid bg + `--color-background` white (`oklch(1 0 0)`) | Page background, iframe surround, toolbar fill |
| Secondary (30%) | `--color-secondary` / `--color-muted` (`oklch(0.97 0 0)`), borders `--color-border` (`oklch(0.922 0 0)` ≈ gray-200), text `--color-foreground` (`oklch(0.145 0 0)` ≈ gray-900) | Toolbar surface, borders, neutral button text, primary CTA fill (near-black `--color-primary`) |
| Accent (10%) — EDIT MODE | **`#2563eb`** (blue-600); tints `#3b82f6` (blue-500), `#eff6ff` (blue-50), border `#bfdbfe` (blue-200) | RESERVED — see list below |
| Destructive | `--color-destructive` (`oklch(0.577 0.245 27.325)` ≈ red-600) | Discard-confirmation dialog emphasis only |

**Accent (edit-mode blue `#2563eb`) reserved for — explicit list:**
1. The active-edit **banner** strip (bg `#eff6ff`, border `#bfdbfe`, text `#1d4ed8`).
2. The **3px border** drawn around the preview iframe while edit mode is active.
3. The in-iframe **selected element** outline (`#2563eb`) and **hover** outline (`#3b82f6`) — applied as inline styles by the edit script.
4. The **dirty-state badge** ("N alterações não salvas") background tint.

Edit-mode blue is NEVER used for the catalog, generic buttons, links, or any non-edit chrome. Outside edit mode the screen is 100% neutral. The primary "Editar" / "Salvar alterações" CTAs use the design-system near-black `--color-primary` (consistent with existing dashboard CTAs), not blue — blue is a *state* signal, not a button color.

## Copywriting Contract

Language: **Portuguese** (matches locked decisions D-01/D-06 and the existing edit flow vocabulary).

| Element | Copy |
|---------|------|
| Primary CTA — enter edit | `Editar` |
| Exit edit, no unsaved changes | `Concluir` |
| Save CTA | `Salvar alterações` |
| Save CTA — pending | `Salvando…` (disabled) |
| Discard CTA | `Descartar` |
| Dirty indicator (plural) | `{N} alterações não salvas` |
| Dirty indicator (singular) | `1 alteração não salva` |
| Active-edit banner | `Modo de edição ativo — clique em um texto para editar` |
| Selection in-progress hint | `Editando texto — Enter para confirmar, Esc para cancelar` |
| Empty/idle (edit mode, nothing selected) | banner copy above (no separate empty state; the banner IS the guidance) |
| Error state — save failed | `Não foi possível salvar as alterações. Tente novamente.` |
| Error state — iframe not ready | `Carregando a preview… aguarde para editar.` (Edit button disabled until `IFRAME_READY`) |
| Destructive confirmation (Discard with unsaved edits) | Title: `Descartar alterações?` · Body: `As {N} alterações não salvas serão perdidas e o texto original será restaurado.` · Confirm: `Descartar` (destructive) · Cancel: `Continuar editando` |

Viewer copy: **none**. Viewers see no edit control, no banner, no hint — the toolbar renders without the "Editar" button (server-gated, `canEdit=false`).

## Component & State Contract (dashboard parent)

### Edit Toolbar (`h-12`, `px-4`, `border-b border-border`, `bg-background`, sticky)

| State | Visible controls | Notes |
|-------|------------------|-------|
| **Viewer (canEdit=false)** | LP name + `Vite SPA` badge + back link only | NO "Editar" button rendered. Server-gated via `can(role,'lp','update')`. |
| **View mode (canEdit, idle)** | `Editar` (primary) | Button disabled until `IFRAME_READY` received. |
| **Edit mode — clean** | `Concluir` (ghost/outline) · dirty indicator hidden | No unsaved edits → `Concluir` exits without confirm. |
| **Edit mode — dirty** | `Descartar` (outline) · `Salvar alterações` (primary) · `{N} alterações não salvas` badge (blue tint) | `Concluir` is replaced by the Save/Discard pair once edits exist. |
| **Saving** | `Descartar` (disabled) · `Salvando…` (primary, disabled, no spinner-only-text) | All controls disabled; iframe not interactive. |

Button mapping to design system: primary actions → `<Button>` default variant (near-black). `Descartar` → `<Button variant="outline">`. `Concluir` → `<Button variant="ghost">`. Dirty count → `<Badge>` with blue-tint className override (`bg-[#eff6ff] text-[#1d4ed8] border-[#bfdbfe]`). All Button and Badge text is `font-semibold` (600) in this phase, overriding the shadcn `font-medium` (500) default.

### Active-Edit Banner (below toolbar)
- Renders ONLY in edit mode. `bg-[#eff6ff] border-b border-[#bfdbfe] text-[#1d4ed8] text-sm h-8 px-4 py-1` (fixed 32px height; 4px vertical padding — no off-grid `py-1.5`).
- Content swaps: idle → `Modo de edição ativo — clique em um texto para editar`; element selected → `Editando texto — Enter para confirmar, Esc para cancelar`.

### Preview Frame Treatment
- View mode: `border: none` (current behavior).
- Edit mode: `outline: 3px solid #2563eb; outline-offset: -3px` on the iframe wrapper, signaling the editable region. Removed immediately on exit/save.

### Discard Confirmation
- Use `<Dialog>`. Trigger ONLY when `Descartar` clicked AND N > 0. If N == 0, `Concluir` exits silently with no dialog.
- Confirm button: `<Button variant="destructive">Descartar</Button>`. Cancel: `<Button variant="ghost">Continuar editando</Button>`.

### Save error
- On `updateLpAction` failure, show inline `<Alert variant="destructive">` above the iframe with the save-failed copy; keep edit mode + pending edits intact (do not discard the user's work).

## In-Iframe Highlight Contract (cross-origin — standalone CSS, NO Tailwind)

These values are applied by the injected edit script via `element.style.*` inline assignments. They are the visual source of truth for what the user sees INSIDE the iframe. They intentionally mirror the edit-mode blue accent but are hard-coded hex (the iframe has no access to app tokens).

| Interaction | CSS (inline, exact) | Trigger |
|-------------|---------------------|---------|
| Hover over text-leaf | `outline: 2px dashed #3b82f6; outline-offset: 2px; cursor: pointer;` | `mouseover` on `isTextLeaf` element, edit mode on |
| Hover clear | `outline: ''; outline-offset: ''; cursor: '';` | `mouseout`, when element is not the selected one |
| Selected element | `outline: 2px solid #2563eb; outline-offset: 2px; background-color: rgba(37,99,235,0.08);` | `click` on text-leaf |
| Active contentEditable affordance | add to selected: `cursor: text; box-shadow: 0 0 0 3px rgba(37,99,235,0.25);` | element becomes `contenteditable=true` + focused |
| Deselect / blur cleanup | clear `outline`, `outline-offset`, `background-color`, `box-shadow`, `cursor`; remove `contenteditable` | blur / Escape / EDIT_MODE_EXIT / discard |

Rules:
- Only **text-leaf** elements (no child elements, non-empty trimmed `textContent`) receive hover/selection treatment (D-03). Skip `script,style,noscript,head,meta,link,br,hr,input,img,svg`.
- The script MUST save and restore each element's pre-existing inline `outline`/`background`/`cursor` (most will be empty string) so removing the highlight does not clobber author styles.
- Highlights must be removed on `EDIT_MODE_EXIT`, `REQUEST_DISCARD`, and before the iframe reloads on save — the served/export HTML must never retain editor styling (SEC-01 spirit; export route never injects the script anyway).
- Do not change layout: use `outline` (not `border`) and `outline-offset` so element box size is unaffected and WYSIWYG fidelity is preserved.

## Extensibility Notes (Phase 11 readiness — D-04)

The toolbar and selection contract are typed for multi-type editing without rework:
- Selection emits `{ path, originalHash, type, ... }`; Phase 10 fixes `type:'text'`. Image (`type:'image'`) and link (`type:'href'`) will reuse the same selection highlight + banner, swapping only the in-context editor surface (upload control / URL input) — a future right-rail or popover keyed off the selected element type.
- The dirty indicator counts overrides regardless of type, so no change is needed when image/link edits are added.
- Reserve a slot in the toolbar (right of `Salvar alterações`) for a future per-type control without reflowing the layout.
