  function pathToNode(path) {
    try {
      var parts = path.split('/').filter(function(p) { return p !== ''; });
      var node = document.body;
      for (var i = 0; i < parts.length; i++) {
        var idx = parseInt(parts[i], 10);
        if (!node || !node.childNodes || isNaN(idx) || idx >= node.childNodes.length) return null;
        node = node.childNodes[idx];
      }
      return node || null;
    } catch(e) { return null; }
  }

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| URL params → serve route | ?edit=1 and ?lpId= are untrusted convenience signals; security gates are HMAC token (already verified) and requireWorkspaceRole in updateLpAction |
| injected edit script → browser DOM | Script runs in the iframe's cross-origin document; postMessage sends use targetOrigin=dashboardOrigin (never '*') |
| iframe postMessage → parent | Parent validates event.origin === serveOrigin before processing any message (Plan 10-03) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-02-01 | Elevation of Privilege | ?edit=1 URL param enabling edit script injection | accept | The edit script enables DOM manipulation but cannot persist anything; persistence requires calling updateLpAction from the parent dashboard, which independently gates on requireWorkspaceRole(["owner","admin","editor"]). A viewer crafting ?edit=1 gets the script injected but has no path to persist edits. |
| T-10-02-02 | Tampering | dashboardOrigin embedding in edit script IIFE | mitigate | dashboardOrigin is set at injection time from process.env.DASHBOARD_ORIGIN (server-side env var), never from URL params. Use JSON.stringify() to embed it as a safe JS string literal — prevents injection if the env value contains quotes or backslashes. Same pattern as escapeJsonForHtml in apply-shim.ts. |
| T-10-02-03 | Information Disclosure | Edit script injected in export ZIP | accept | Export route is a separate handler (apps/web/src/app/api/lps/[lpId]/export/route.ts); it never reads ?edit=1 URL param and does not call injectEditScript. The serve route's editMode check only runs in the serve route handler. SEC-01 principle upheld without additional code. |
| T-10-02-04 | Tampering | ?lpId= URL param used as WHERE clause value | mitigate | landingPage.findUnique WHERE clause includes `workspaceId` from HMAC token claims (never from URL). A forged lpId for a different tenant's LP returns null (Prisma returns null on workspace mismatch), not a cross-tenant record. workspaceId is the authoritative tenant scope. |
| T-10-02-05 | Stored XSS | edit script injected into served HTML | mitigate | The injected IIFE applies text edits exclusively via node.textContent (NEVER innerHTML), same control as T-09-02-02. dashboardOrigin is embedded via JSON.stringify (not raw string concatenation). The IIFE does not eval() any data from the DOM or postMessage. |
</threat_model>
