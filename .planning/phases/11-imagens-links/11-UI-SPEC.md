## Component & State Contract (dashboard parent extension)

### Extended postMessage Protocol

Phase 10 postMessage types: `IFRAME_READY`, `EDIT_MODE_ENTER`, `ELEMENT_SELECTED`, `ELEMENT_CHANGED`, `ELEMENT_REVERTED`, `REQUEST_SAVE`, `PENDING_EDITS`, `REQUEST_DISCARD`, `EDIT_DISCARDED`.

Phase 11 extends the protocol as follows:

**Extended (shape change):**

`ELEMENT_SELECTED` (iframe → parent) — extends existing type to carry element type and current value:

```ts
{
  type: "ELEMENT_SELECTED"
  path: string              // unchanged
  elementType: "text" | "image" | "href"  // NEW
  currentValue: string      // NEW — current src (image) or href (link) or empty (text)
}
```

**New (parent → iframe):**

```ts
{
  type: "PREVIEW_OVERRIDE"
  path: string
  elementType: "image" | "href"
  value: string   // new src or new href — applied visually without saving
}
```

Sent by the parent after the user confirms an image or href override, so the iframe reflects the change immediately (visual preview before server save). The iframe applies this via attribute assignment (`img.src = value` / `a.href = value`), never via innerHTML.

**New (iframe → parent):**

No additional iframe→parent messages needed for Phase 11.

### `ImageSwapPanel` Component

Rendered as a `Popover` anchored to the `Trocar imagem` trigger button in the D-04 slot.

**Props:**
```ts
interface ImageSwapPanelProps {
  slug: string
  currentSrc: string          // pre-fills URL input with current image src
  onConfirm: (value: string) => void  // called with final URL when user applies
  open: boolean
  onOpenChange: (open: boolean) => void
}
```

**Internal state:** `uploadState: 'idle' | 'uploading' | 'uploaded' | 'error'`, `uploadProgress: number`, `uploadedUrl: string | null`, `urlValue: string`, `urlError: string | null`.

**Layout inside Popover (w-80 / 320px, `p-4` / 16px padding):**

```
[ Panel header: "Trocar imagem" (text-sm font-semibold) ]

[ Upload zone (h-20 / 80px) ]
  idle:      dashed border + UploadCloud icon + "Arraste uma imagem ou clique para fazer upload" (text-sm) + "PNG, JPG, WEBP · Máx 5 MB" (text-xs text-muted-foreground)
  drag-over: border-blue-300 bg-blue-50
  uploading: Progress bar (h-1.5) + "Enviando… {N}%" + Cancel button
  uploaded:  green-50 bg, 48×48 thumbnail, "Imagem enviada" (text-sm font-medium text-gray-900), Remove button (X icon)
  error:     red-50 bg, AlertCircle icon, errorMessage (text-sm text-red-600), "Tentar novamente" link

[ Separator: 1px line + "ou" centered (text-xs text-muted-foreground) ]

[ Input — URL da imagem (h-9) ]
  placeholder: "https://exemplo.com/imagem.jpg"
  border-destructive + error <p> (text-xs text-destructive mt-1) when urlError set

[ Spacer: mt-4 (16px) ]

[ "Aplicar imagem" Button — variant="default" — 100% width (w-full) ]
  disabled when: uploadState === 'uploading' || (uploadState !== 'uploaded' && !isUrlValid)
  Tooltip wraps disabled state: "Envie um arquivo ou insira uma URL válida para aplicar"
```

**Confirm logic:** "Aplicar imagem" resolves the final URL as follows:
- If `uploadState === 'uploaded'`: use `uploadedUrl` (S3 public URL).
- Else if `urlValue` is non-empty and valid: use `urlValue`.
- After confirming: call `onConfirm(finalUrl)` → parent adds override `{ type: 'image', path, originalHash, value: finalUrl }` to pendingEdits → sends `PREVIEW_OVERRIDE` to iframe → closes Popover.

**When the Popover closes without confirming:** any in-progress upload is cancelled (abort XHR); no override is created; `urlValue` is discarded. The element remains selected (selectedElementType stays 'image'); re-opening the Popover starts fresh.

**Upload reuse:** reuse the S3 upload logic from `apps/web/src/lib/project-templates/s3-upload.ts` and the `requestPresignedUploadAction` / `validateUploadedImageAction` Server Actions (same as `ImageUploadField.tsx`). Do NOT wrap in RHF Controller — this component manages its own state directly (not backed by a form).

### Href URL Input (toolbar inline)

Rendered directly in the D-04 toolbar slot when `selectedElementType === 'href'`.

**Layout:**

```
[ Input (w-60 / 240px, h-9 / 36px) ]  [ optional: confirm hint ]
  type="url"
  aria-label="Destino do link (href)"
  placeholder="https://..."
  value={hrefValue}
  onChange → update state + run client validation → set hrefInputError
  onKeyDown(Enter) → if valid: confirm override; if invalid: do nothing (keep error shown)
  onBlur → if non-empty and valid: confirm override
  border-destructive when hrefInputError is set

[ Tooltip wrapping the Input — shows hrefInputError when set ]
```

**Confirm logic (on Enter or blur with valid URL):**
- Add override `{ type: 'href', path, originalHash, value: hrefValue }` to pendingEdits.
- Send `PREVIEW_OVERRIDE { path, elementType: 'href', value: hrefValue }` to iframe.
- Clear `hrefInputError`.
- Do NOT close or hide the input — it stays visible while the element remains selected.

**Revert on Escape:** if user presses Escape while the href input is focused:
- Revert `hrefValue` to `selectedCurrentValue` (original href).
- Remove any pending override for this path from pendingEdits.
- Clear `hrefInputError`.

## URL Validation Contract (D-11-03)

Both the image URL field and the href input share the same client-side validation logic. This must be implemented as a shared utility (e.g. `apps/web/src/lib/overrides/validate-url.ts`).

**Client validation (synchronous — instant UX feedback, NOT the trust boundary):**

```ts
type UrlValidResult =
  | { ok: true }
  | { ok: false; error: "invalid-protocol" | "malformed" }

function validateOverrideUrl(raw: string): UrlValidResult {
  if (!raw) return { ok: true }  // empty = no error (field not yet filled)
  let parsed: URL
  try { parsed = new URL(raw) } catch { return { ok: false, error: "malformed" } }
  if (parsed.protocol !== "http:" && parsed.protocol !== "https:") {
    return { ok: false, error: "invalid-protocol" }
  }
  return { ok: true }
}
```

Error → copy mapping:
- `"invalid-protocol"` → `"URL inválida — apenas http:// e https:// são permitidos."` (image panel) / `"Apenas http:// e https:// são permitidos."` (href toolbar, shorter for space)
- `"malformed"` → `"URL inválida — verifique o endereço."` (image panel) / `"URL inválida."` (href toolbar)

**Server validation (authoritative — runs in `updateLpAction` before any persist):**

Runs the same protocol allowlist check for all overrides where `type === 'image' || type === 'href'`. A URL that passes client validation can still be rejected server-side (e.g. race condition or tampered request). On server rejection, `updateLpAction` returns `{ ok: false, error: "..." }` and the existing `saveError` Alert is shown above the iframe (inherited behavior — no new error surface needed).

Blocked: `javascript:`, `data:`, `vbscript:`, and any non-`http:`/`https:` protocol. Malformed URLs (fail `new URL()` on the server) are also rejected. No override with an invalid URL is persisted.

## In-Iframe Highlight Contract (Phase 11 additions)

All values applied via `element.style.*` inline assignments by the extended edit script. Phase 10 values (text-leaf hover/selected/contenteditable) are inherited unchanged.

### New: `<img>` elements

| Interaction | CSS (inline, exact) | Trigger |
|-------------|---------------------|---------|
| Hover over `<img>` | `outline: 2px dashed #3b82f6; outline-offset: 2px; cursor: pointer;` | `mouseover` on `<img>`, edit mode on |

| Hover over `<a href>` | `outline: 2px dashed #3b82f6; outline-offset: 2px; cursor: pointer;` | `mouseover` on `<a[href]>`, edit mode on |

| Deselect `<a>` | clear `outline`, `outline-offset`, `background-color`, `cursor` | EDIT_MODE_EXIT / new selection |

### Precedence rule (click target hierarchy)

When `<img>` is a child of `<a>` and the user clicks:
- Clicking the `<img>` → selects **image** (innermost match wins).
- Clicking the `<a>` area outside its `<img>` child → selects **href**.
- Text nodes inside `<a>` → selects **text** (text-leaf rule, unchanged from Phase 10).

### Rules (additions to Phase 10 rules)

- `<img>` and `<a[href]>` are selectable in edit mode. Bare `<a>` without an `href` attribute is NOT selectable.
- NO `contenteditable` is set on `<img>` or `<a>` elements. The content-editable affordance remains exclusive to text-leaf elements.
- Use `outline` (not `border`) to preserve element box size and WYSIWYG fidelity — unchanged rule from Phase 10.
- Save and restore each `<img>` / `<a>` element's pre-existing inline `outline`, `background-color`, and `cursor` before applying editor styles.
- All highlights removed on `EDIT_MODE_EXIT`, `REQUEST_DISCARD`, and before iframe reloads — the served/export HTML must never retain editor styling.
- After `PREVIEW_OVERRIDE` is applied for an image: keep the selection outline on the `<img>` (element remains selected); update the visual `src` only.
