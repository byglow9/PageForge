## Spacing Scale

Declared values (multiples of 4). All values carried forward from Phase 1-4 patterns.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding between icon and label |
| sm | 8px | Compact element spacing, menu item padding top/bottom |
| md | 16px | Default element spacing, form field gaps |
| lg | 24px | Section padding, card grid gap |
| xl | 32px | Layout column/row gaps |
| 2xl | 48px | Major section breaks (page-level vertical rhythm) |
| 3xl | 64px | Not used in dashboard; reserved for landing page output |

Page content padding: `px-8 py-6` (32px horizontal, 24px vertical) — matches existing lps/templates pages.

Sidebar width: 240px (`w-60`) — matches existing workspace layout.

Folder tree indent: 16px per nesting level.

Exceptions: none.

## Typography

Inherited directly from Phase 1-4 established patterns (globals.css + existing pages).

Two weights only: 400 (normal) and 600 (semibold). No other weights are used anywhere in this phase.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px (text-sm) | 400 (normal) | 1.5 | Metadata, card body text, form labels, search input |
| Label / UI | 14px (text-sm) | 600 (semibold) | 1.4 | Nav links, badge text, menu items |
| Heading | 24px (text-2xl) | 600 (semibold) | 1.2 | Page-level headings (e.g. "Catalog") |
| Card Title | 16px (text-base) | 600 (semibold) | 1.3 | LP card name, folder card name |

Muted text: text-sm (14px) weight 400, color `--color-muted-foreground` (`oklch(0.556 0 0)`).

## Component Inventory

### New components for Phase 5

All new components follow the existing shadcn/Tailwind conventions established in Phases 3-4.

| Component | Type | Description |
|-----------|------|-------------|
| `FolderTree` | Client component | Left-panel nested folder list with expand/collapse. Active folder highlighted with `bg-white` + `text-gray-900 font-semibold`. Each folder row: `h-8`, indent 16px per depth level, chevron icon (ChevronRight / ChevronDown 14px). Root = "All LPs" always present. |
| `FolderContextMenu` | Client component | Right-click or kebab (MoreHorizontal) menu on folder rows: "New subfolder", "Rename", separator, "Delete folder". Uses same custom-positioned `div` menu pattern as existing TemplateCard/LpCard kebab. |
| `CreateFolderDialog` | Client component | shadcn `Dialog`. Single text input for folder name. CTAs: "Cancel" (outline) + "Create folder" (default). |
| `RenameFolderDialog` | Client component | shadcn `Dialog`. Single text input pre-filled with current name. CTAs: "Cancel" (outline) + "Rename folder" (default). |
| `DeleteFolderDialog` | Client component | shadcn `Dialog`. Warns about LP re-placement to root. CTAs: "Keep folder" (outline) + "Delete folder" (destructive). |
| `TagInput` | Client component | Inline tag chip input on LP detail/edit surface. Uses shadcn `Badge` (variant: secondary) for rendered tags. Add tag: press Enter or comma. Remove tag: × on chip. Max tag length: 32 chars. |
| `CatalogSearchBar` | Client component | `Input` (full-width, search icon left). Filters visible LPs by name and tags client-side (no API call on keystroke). Debounce: none needed for client-side filter; instant. Placeholder: "Search landing pages…". |
| `CatalogFilterBar` | Client component | Horizontal pill row below search. "All" pill (default active), one pill per unique tag in the workspace. Active pill: `bg-gray-900 text-white`. Inactive pill: `border border-gray-200 bg-white text-gray-700`. |
| `LpCatalogCard` | Extends `LpCard` | Adds: folder badge (tag-style, gray), category tag chips (up to 3 visible + "+N more"). Otherwise identical layout to existing LpCard. |

### Reused shadcn components (already installed)

badge, button, card, dialog, input, label, separator, skeleton, sonner, tooltip.

### New shadcn components to install

`dropdown-menu` — to replace the custom-positioned kebab `div` pattern on folder rows (FolderContextMenu), making it keyboard-accessible without re-implementing focus trapping.

No other net-new installs required. The custom kebab pattern on LpCard and TemplateCard is retained for backward consistency; new folder context menus use DropdownMenu.

## Interaction Contracts

### Folder creation
- Trigger: "New folder" button at top of FolderTree (plus `+` icon, text-sm) or from FolderContextMenu "New subfolder".
- Dialog: CreateFolderDialog. Name is required, max 64 chars. Empty name: inline validation "Folder name is required."
- On success: folder appears in tree, expanded if it has a parent, toast "Folder created."
- On error: toast "Failed to create folder. Try again."

### Folder rename
- Trigger: FolderContextMenu "Rename".
- Dialog: RenameFolderDialog. Pre-filled. Same validation as create.
- On success: tree label updates, toast "Folder renamed."

### Folder deletion
- Trigger: FolderContextMenu "Delete folder."
- Dialog: DeleteFolderDialog. Body copy: "This will delete the folder. Landing pages inside will be moved to the root catalog."
- CTAs: "Keep folder" (outline) + "Delete folder" (destructive).
- On success: tree removes folder, LPs re-appear under "All LPs", toast "Folder deleted."

### Moving an LP to a folder
- Trigger: LpCatalogCard kebab menu "Move to folder…"
- Interaction: inline sub-menu or a Dialog listing the folder tree (flat with indent). Select folder and confirm.
- On success: LP disappears from current folder view (if filtered), toast "Moved to [folder name]."

### Tag management on LP
- Surface: LpCatalogCard kebab "Edit tags…" OR inline on the LP edit page.
- Interaction: TagInput component.
- Tags saved on blur / Enter. Persisted via a server action.
- On success: chips update in place, no toast (inline feedback sufficient). On error: toast "Failed to save tags."

### Search and filter
- SearchBar filters the visible LP grid client-side by LP name substring (case-insensitive) and tag match.
- FilterBar pills filter by single tag (click to activate, click again to deactivate — toggles).
- Search + FilterBar are AND-combined: matching LPs must satisfy both the text query and the selected tag filter simultaneously.
- While no filter is active: "All" pill is visually active (`bg-gray-900 text-white`).
- Zero results: inline empty state within the LP grid area (not full-page): centered text "No landing pages match your search." No icon. No CTA.

### Grécia acceptance loop
- No new UI surface needed. Uses the existing LP generate → preview → edit → duplicate → export flow from Phase 4.
- Acceptance is verified manually via the UAT checklist in the verification report.

## Accessibility

- All interactive folder rows are `<button>` elements (not `<div>` with onClick).
- FolderContextMenu DropdownMenu: keyboard-navigable (shadcn DropdownMenu uses Radix Popover + keyboard handling out of the box).
- Search input: `aria-label="Search landing pages"`.
- FilterBar pills: `role="radio"` group or `aria-pressed` per pill (single-select toggle semantics).
- Tag chips: remove button `aria-label="Remove tag [name]"`.
- Folder expand/collapse chevron: `aria-expanded={boolean}` on the parent button.
- All dialogs use shadcn Dialog (Radix focus trap + aria-modal).
