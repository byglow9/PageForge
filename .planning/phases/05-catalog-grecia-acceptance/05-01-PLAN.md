<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client browser → Server Action | Folder names, parentId, folderId, lpId, tagNames arrive as untrusted client input |
| Server Action → withTenantDb | workspaceId must come from server session (requireWorkspace/requireWorkspaceRole), never from client |
| Folder move → cycle check | parentId on folder move must not create a cycle (a folder cannot be its own ancestor) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01-01 | Spoofing/IDOR | createFolderAction, renameFolderAction, deleteFolderAction | mitigate | workspaceId always from requireWorkspaceRole; db.folder.findById always filters by workspaceId — cross-workspace folderId returns null |
