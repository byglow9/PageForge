<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → saveBrandConfigAction | logoUrl (URL), primaryColor (hex), whatsapp (free text) cross trust boundary |
| Server Action → TenantClient | workspaceId from session only; brandConfig.upsert uses @unique workspaceId constraint |
| BrandConfig → Phase 4 render() | logoUrl injected as brand.logo into LP HTML at render time — Phase 4 must sanitize |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04-01 | Information Disclosure | getBrandConfigAction / brand page | mitigate | withTenantDb sets app.current_workspace_id; brandConfig.findFirst() includes workspaceId in WHERE; RLS backstop prevents cross-workspace read |
| T-03-04-02 | Elevation of Privilege | viewer calling saveBrandConfigAction | mitigate | requireWorkspaceRole(slug, ["owner","admin","editor"]) redirects viewer before any DB write |
| T-03-04-03 | Tampering | primaryColor CSS injection at render time | mitigate (Phase 4) | Zod validates as /^#[0-9a-fA-F]{6}$/ at save boundary — only valid hex stored. Phase 4 must not inject raw value into style attributes without CSS sanitization; for now (Phase 3) color is only stored and displayed in token reference block as text. |
| T-03-04-04 | Tampering | logoUrl open redirect / XSS at render time | mitigate (Phase 4) | Zod validates https:// scheme at save boundary. Phase 4 must sanitize before injecting as src attribute in LP HTML (RESEARCH.md known threat). Phase 3 only stores the URL. |
| T-03-04-05 | Tampering | whatsapp field injection | accept | Free text, max 32 chars (Zod). No format validation in v1 (D-07). The field is used as contact info in LPs — Phase 4 must HTML-escape before injection. Low risk at Phase 3 (storage only). |
| T-03-04-06 | Denial of Service | Multiple upsert race conditions (two concurrent saves) | accept | @unique on workspaceId + Prisma upsert is atomic at DB level. Worst case: last write wins. Acceptable for v1 single-user brand config. |
| T-03-04-07 | Information Disclosure | brand token reference block reflects stored values | accept | Block shows the stored brand values back to the authenticated editor who owns the workspace. No cross-workspace disclosure. Values are user-entered; reflected in same session. |
</threat_model>
