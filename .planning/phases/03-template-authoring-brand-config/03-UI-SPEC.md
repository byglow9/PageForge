## Spacing Scale

Declared values (all multiples of 4 — 8-point grid):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding, tight metadata rows |
| sm | 8px | Input internal padding, compact button padding, card inner gaps |
| md | 16px | Default element spacing, form field gaps, card padding |
| lg | 24px | Section padding, nav padding, editor toolbar gaps |
| xl | 32px | Layout content area padding, panel gaps |
| 2xl | 48px | Major section breaks, page-level vertical rhythm |
| 3xl | 64px | Full-page centering blocks (auth/empty states) |

Exceptions:
- Touch targets for icon-only buttons (schema panel action icons): minimum 44px hit area (WCAG 2.5.5). Achieved via `p-2.5` on a 24px icon = 44px target.
- Workspace nav bar height: 48px (`py-3 px-6`) — matches the existing Phase 2 inline style (`padding: "0.75rem 1.5rem"`) which resolves to this scale.
- Split-pane editor gutter (divider drag handle): 4px visual width, 16px hit area.

## Typography

| Role | Size | Weight | Line Height | Tailwind class | Usage |
|------|------|--------|-------------|----------------|-------|
| Body | 14px | 400 (regular) | 1.5 | `text-sm` | Field labels, metadata rows, table cells, helper text, badges, version badges, section headings (schema panel), monospace code blocks |
| Label/UI | 16px | 600 (semibold) | 1.5 | `text-base font-semibold` | Form labels, nav items, card titles, button text, template name input |
| Heading | 20px | 600 (semibold) | 1.25 | `text-xl font-semibold` | Page section headings (Templates, Brand Settings) |
| Display | 28px | 600 (semibold) | 1.2 | `text-2xl font-semibold` | Page-level H1 (e.g. "Templates") |

Font weight palette: **400 (regular)** for body/content and **600 (semibold)** for all emphasis (headings, display, labels, nav items, button text, card titles). No other weights used.

Code/monospace: `font-mono text-sm` — applied exclusively to the markup editor textarea and token examples (e.g. `{{ field:text }}`).

## Color

All values are Tailwind gray scale neutrals — consistent with existing Phase 2 inline styles.

| Role | Tailwind Token | Hex | Usage |
|------|---------------|-----|-------|
| Dominant (60%) | `gray-50` / `white` | `#f9fafb` / `#ffffff` | Page background (`gray-50`), card/panel surfaces (`white`) |
| Secondary (30%) | `gray-100` | `#f3f4f6` | Sidebar/nav background, schema side panel background, input prefix blocks, role badges |
| Accent (10%) | `gray-900` | `#111827` | Primary CTA buttons, active nav item, focus ring border |
| Destructive | `red-600` | `#dc2626` | Delete template confirmation button only |

Accent reserved for:
1. Primary action buttons ("Save Template", "Create Template", "Save Brand Settings")
2. Active/selected nav item indicator (left border or background pill)
3. Focus ring on interactive elements (via Tailwind `ring-gray-900`)

Accent is NOT used for: links, secondary buttons, status badges, parse warning chips, info callouts, or hover states.

Secondary semantic colors (not accent, not destructive):
- Parse warning chip: `amber-100` background, `amber-800` text — reserved exclusively for parse warnings in the schema side panel
- Parse error chip: `red-100` background, `red-700` text — reserved for hard parse errors (unclosed repeaters)
- Field type badge: `blue-50` background, `blue-700` text — field type labels in the schema panel
- Repeater badge: `purple-50` background, `purple-700` text — repeater block indicators in the schema panel
- Global/brand badge: `green-50` background, `green-700` text — `brand.*` token indicators in the schema panel
- Success toast: `green-50` background, `green-800` text — save confirmation

## Page Layouts

### TPL-06 — Template List (`/w/[slug]/templates`)

**Focal point:** "Create Template" accent CTA in the empty state; the template card grid in the populated state.

Layout: 2-column workspace shell.
- Left: persistent workspace sidebar (nav links: Templates, Brand Settings, Members; 240px wide, `gray-100` background).
- Right: main content area (`gray-50` background, `px-8 py-6`).

Template grid: CSS Grid, `grid-cols-1 sm:grid-cols-2 lg:grid-cols-3`, `gap-6`.

Each template card (shadcn `<Card>`):
- Header: template name (`text-base font-semibold`) + schema version badge (`text-sm text-gray-400`)
- Body: field count summary ("5 fields · 1 repeater") as `text-sm text-gray-500`
- Footer: "Edit Template" button (secondary/outline) + "..." overflow menu (kebab icon, lucide `MoreHorizontal`)
- Card width: fills grid column. Min height: 120px.

Empty state (no templates yet):
- Centered in content area, vertically centered in remaining viewport height.
- Illustration: lucide `FileCode` icon at 48px, `text-gray-300`.
- Heading: "No templates yet" (`text-xl font-semibold text-gray-900`).
- Body: "Create your first template to start building landing pages." (`text-sm text-gray-500`).
- CTA: "Create Template" button (primary/accent).

### TPL-01 / TPL-05 — Template Editor (`/w/[slug]/templates/new` and `/w/[slug]/templates/[id]/edit`)

**Focal point:** the markup editor pane (`flex-1`, dominant area).

Layout: full-width within workspace shell. Two panels side by side.

```
┌─────────────────────────────────────────────────────────────────┐
│  Nav: PageForge / workspace-slug            [role badge]        │
├──────────────────────────────────────────────────────────────────┤
│  Sidebar (240px)     │  Editor Area (flex-1)                    │
│  - Templates         │  ┌──────────────────┬──────────────────┐ │
│  - Brand Settings    │  │  Markup Editor   │  Schema Panel    │ │
│  - Members           │  │  (textarea, mono)│  (detected fields│ │
│                      │  │                  │  + warnings)     │ │
│                      │  │                  │                  │ │
│                      │  └──────────────────┴──────────────────┘ │
│                      │  [Cancel]              [Save Template]   │
└──────────────────────┴──────────────────────────────────────────┘
```

- Editor panel: `flex-1`, `min-h-[500px]`, `font-mono text-sm`, no syntax highlighting in v1 (plain `<Textarea>` with resize-y disabled, `h-full`).
- Schema panel: `w-80` (320px), `gray-50` background, `border-l border-gray-200`, scrollable independently.
- Divider between panels: 1px `gray-200` border, no drag in v1.
- Template name input: above the two panels, full-width, `text-base font-semibold` (16px, Label/UI role), placeholder "Template name".
- Toolbar row (above name input): breadcrumb (`Templates / New Template`) + save button aligned right.

Schema panel internals:
- Section heading "Detected Fields" (`text-sm font-semibold text-gray-500 uppercase tracking-wide`).
- Per-field row: field name (`text-sm font-semibold`) + type badge (color-coded per field type) + optional "brand" or "global" badge.
- Parse warning rows below detected fields (amber badge + message text `text-sm`).
- Live update: debounced 400ms after the user stops typing. Show spinner (`lucide Loader2`, animate-spin) in panel header while debounce is pending.
- "Re-parse" icon button at panel top right (`lucide RefreshCw`) forces immediate re-parse.
- If no fields detected: "No tokens found yet. Use `{{ field:type }}` syntax." in `text-sm text-gray-400 italic`.

Metadata overlay (below detected fields in schema panel):
- Collapsed by default. "Edit field metadata" toggle link (`text-sm text-gray-500 underline`).
- When expanded: per-field sub-row with a label input (`text-sm`, placeholder = field name) and a `<Switch>` labeled "Required".
- Metadata rows are reconciled on every save (D-05).

### BRD-01 — Brand Settings (`/w/[slug]/brand`)

**Focal point:** the single centered settings card with its full-width "Save Brand Settings" primary button.

Layout: workspace shell sidebar + main content.

Form is a single `<Card>` centered, max-width 560px, `mx-auto mt-8`.

Fields (in order):
1. **Logo URL** — `<Input>` type="url", label "Logo URL", helper "Paste a publicly accessible image URL. Image upload coming later.", placeholder "https://example.com/logo.png"
2. **Primary Color** — `<Input>` type="text", label "Primary Color", placeholder "#0f172a", helper "Hex color. Used in templates via `brand.primary_color`."
3. **WhatsApp / Contact** — `<Input>` type="text", label "WhatsApp number", placeholder "+55 11 99999-9999", helper "Used in templates via `brand.whatsapp`."

Color preview: 24px × 24px swatch inline with the Primary Color input, updated live on valid hex input. Shows `gray-200` background when empty/invalid.

Brand token reference block (below form fields, above submit): gray code block listing the three `brand.*` tokens with their resolved values. `font-mono text-sm text-gray-600 bg-gray-50 rounded p-3 border border-gray-200`.

Submit: "Save Brand Settings" button (primary, full-width of card).

## Interaction Contracts

### Save Template flow (TPL-01 / TPL-05)
1. User clicks "Save Template".
2. Button enters pending state: `disabled`, label "Saving…", lucide `Loader2` spinner left of label.
3. Server Action runs: parse markup → derive schema → reconcile metadata overlay → upsert Template row (increment `schema_version`).
4. If parse produces warnings: show `<Alert variant="warning">` above the editor with warning list. Template is still saved (tolerant parse, D-03).
5. If parse produces hard errors (e.g. unclosed repeater): show `<Alert variant="destructive">`. Template is still saved with warnings in schema.
6. On success: `<Sonner>` toast "Template saved — schema v{N}" slides in bottom-right. Schema panel refreshes to show save-time schema.
7. On server error (network/DB): toast "Failed to save. Try again." (destructive). Button re-enables.

### Live parse feedback
- Debounce: 400ms after last keystroke.
- While pending: schema panel header shows `Loader2` spinner.
- On result: panel updates field list + warnings without full-page reload.
- Live parse is advisory. Save-time parse is authoritative (D-02).

### Create Template flow
- Route: `/w/[slug]/templates/new`
- Template name field is required. Blur validation: "Template name is required." in `text-sm text-red-600 mt-1`.
- Save is disabled until name field has at least 1 non-whitespace character.

### Delete Template
- Trigger: kebab menu item "Delete template" on template card.
- Opens `<Dialog>`: title "Delete template?", body "This will permanently delete "[template name]" and cannot be undone.", destructive button "Delete template" (`variant="destructive"`), cancel button "Keep template".
- On confirm: server action fires, card removed optimistically from grid, toast "Template deleted."
- On error: toast "Failed to delete. Try again." Dialog stays open.

### Brand Settings save
- Single "Save Brand Settings" button.
- Client-side: primary_color validated as valid hex on blur (`/^#[0-9a-fA-F]{6}$/`) — inline error "Enter a valid 6-digit hex color (e.g. #0f172a)".
- Logo URL validated as valid URL on blur — inline error "Enter a valid URL starting with https://".
- WhatsApp: no client-side format validation in v1 (free text).
- On save: toast "Brand settings saved."
- On server error: toast "Failed to save brand settings. Try again."

## Accessibility

- All interactive elements have visible focus rings (Tailwind `focus-visible:ring-2 ring-gray-900 ring-offset-2`).
- Alert components use `role="alert"` (shadcn `<Alert>` provides this).
- Dialog uses `role="dialog"` with `aria-labelledby` and `aria-describedby` (shadcn `<Dialog>` provides this).
- Schema panel parse results: `aria-live="polite"` region so screen readers announce field count changes.
- Color swatch in brand form: `aria-label="Color preview"`, `role="img"`.
- Icon-only buttons (kebab, re-parse, schema panel actions): `aria-label` required on every instance.
- Required field switches in metadata overlay: `aria-label="{field name} required"`.
- Minimum color contrast: all body text on white/gray-50 surfaces passes WCAG AA (gray-600 `#4b5563` on white = 7.2:1).
