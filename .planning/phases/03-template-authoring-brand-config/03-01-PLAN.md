<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client→Server Action | Markup content, template name, brand fields cross from browser to server |
| server→DB | workspaceId must come from session, not client; RLS as DB-level backstop |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01-01 | Information Disclosure | TenantClient.template.findById | mitigate | findFirst uses { id, workspaceId } — cross-workspace ID lookup returns null at app layer; RLS backstop when workspaceId is SET LOCAL |
| T-03-01-02 | Tampering | Prisma schema / RLS missing | mitigate | Plan 02 appends RLS SQL to migration; schema-conventions test enforces workspaceId presence on all tenant models |
| T-03-01-03 | Elevation of Privilege | workspaceId from client | mitigate | TenantClient is only constructable inside withTenantDb; workspaceId always comes from requireWorkspace/requireWorkspaceRole server-side context |
| T-03-01-04 | Tampering | SaveBrandConfigSchema — primaryColor CSS injection | mitigate | Zod regex /^#[0-9a-fA-F]{6}$/ at server action boundary; no CSS injection risk in Phase 3 (color only stored, not rendered) |
| T-03-01-05 | Tampering | SaveBrandConfigSchema — logoUrl server-side open redirect | mitigate | z.string().url().startsWith("https://") validates scheme; Phase 4 must also sanitize before injecting into LP HTML |
| T-03-01-06 | Denial of Service | markup String @db.Text unbounded size | accept | MVP v1 — no size limit on markup. Risk: large markup inflates DB and parse time. Acceptable for internal authoring tool with RBAC-gated authors. Add max length in Phase 4+ if needed. |
| T-03-01-07 | Tampering | renderer.ts pulled into client bundle via parse import | mitigate | "sideEffects": false on engine package.json + named import { parse } from "pageforge-engine" (never { render }) in client components; pitfall documented in RESEARCH.md |
</threat_model>
