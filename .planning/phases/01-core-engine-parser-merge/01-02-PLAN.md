<threat_model>
## Trust Boundaries

| Boundary | Descrição |
|----------|-----------|
| markup do autor → parser | Template HTML com tokens — pode conter tokens malformados, mas não valores de usuário |
| values do chamador → renderer | Dados preenchidos pelo usuário — UNTRUSTED; cada campo tem contexto de escaping diferente |
| LiquidJS scope → HTML output | LiquidJS processa o template compilado com os valores pré-processados como dados |

## STRIDE Threat Register (Plano 02 — Parser + Compiler + Renderer)

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | LiquidJS config (ownPropertyOnly) | mitigate | `ownPropertyOnly: true` configurado explicitamente em renderer.ts (D-10); nunca usar false; verificado em acceptance_criteria por grep |
