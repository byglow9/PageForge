## Spacing Scale

Inherited from Phase 3 (8-point grid). No changes for Phase 4.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding, field type chips |
| sm | 8px | Input internal padding, compact button padding, repeater item inner gaps |
| md | 16px | Default element spacing, form field gaps, card padding |
| lg | 24px | Section padding, repeater group headers, form section separators |
| xl | 32px | Layout content area padding, page-level panel gaps |
| 2xl | 48px | Major section breaks (form header vs. first field group) |
| 3xl | 64px | Empty/loading states vertical centering |

Exceptions:
- Touch targets for icon-only add/remove repeater buttons: minimum 44px hit area (WCAG 2.5.5) via `p-2.5` on a 20px icon.
- Preview iframe: no padding — fills its container edge-to-edge so the LP renders at its natural dimensions.
- Image upload drop zone: `min-h-[128px]` (32 × 4 = 128px) for a usable drag target.

## Color

Inherited from Phase 3 (neutral gray palette). No new color roles added. Extended with upload-specific and preview-specific states.

| Role | Tailwind Token | Approx Hex | Usage |
|------|---------------|------------|-------|
| Dominant (60%) | `gray-50` / `white` | `#f9fafb` / `#ffffff` | Page background (`gray-50`), card/form surfaces (`white`) |
| Secondary (30%) | `gray-100` | `#f3f4f6` | Sidebar/nav background, repeater group tint, brand globals read-only block, upload drop zone idle background |
| Accent (10%) | `gray-900` | `#111827` | Primary CTA buttons ("Generate LP", "Export ZIP", "Save Changes"), active nav item |
| Destructive | `red-600` | `#dc2626` | Delete LP confirmation button, duplicate-overwrite confirmation only |

Accent reserved for:
1. Primary action buttons: "Generate LP", "Export ZIP", "Save Changes" (edit mode), "Duplicate LP"
2. Active/selected nav item indicator
3. Focus ring on interactive elements

Accent is NOT used for: preview toggle buttons, add/remove repeater item buttons, export download link, secondary actions, status chips, or hover states.

Secondary semantic colors (unchanged from Phase 3, extended):
- Upload progress fill: `blue-500` progress bar fill on `gray-200` track — upload progress only
- Upload success state: `green-50` background, `green-700` border, `green-700` text — image upload confirmed
- Upload error state: `red-50` background, `red-300` border, `red-600` text — file validation failure
- Validation error inline: `red-600` text — required field not filled
- Brand globals display block: `gray-50` background, `gray-200` border, `gray-600` text — read-only brand token values in the form
- Schema version outdated chip: `amber-100` background, `amber-800` text — displayed when LP's snapshot schema_version < template's current schema_version (D-08)
- LP action badges: same pattern as Phase 3 badges (variant="outline" for status, variant="secondary" for role)

## Page Layouts

### GEN-01 — LP List (`/w/[slug]/lps`)

**Focal point:** "Generate LP" accent CTA in the empty state; the LP card grid in the populated state.

Layout: 2-column workspace shell (identical to templates page — sidebar 240px + main content area).

Sidebar navigation: add "Landing Pages" link between "Templates" and "Brand Settings" in the workspace sidebar (`/w/[slug]/lps`).

LP grid: CSS Grid, `grid-cols-1 sm:grid-cols-2 lg:grid-cols-3`, `gap-6`.

Each LP card (`<Card>`):
- Header: LP name (`text-base font-semibold`) + template name source in muted text (`text-sm text-gray-400 font-normal`, e.g. "from Grécia v2")
- Body: generated/last-edited date (`text-sm text-gray-500`)
- Footer: "Preview" button (secondary/outline) + "Edit" button (secondary/outline) + kebab overflow menu (`MoreHorizontal`, lucide) with: "Duplicate", "Export ZIP", separator, "Delete landing page"
- Card min-height: 120px

Empty state (no LPs yet):
- Centered in content area, vertically centered.
- Icon: lucide `FileText` at 48px, `text-gray-300`.
- Heading: "No landing pages yet" (`text-xl font-semibold text-gray-900`).
- Body: "Pick a template and fill in the form to generate your first landing page." (`text-sm text-gray-500`).
- CTA: "Generate LP" button (primary/accent) — links to template picker.

### GEN-01 — Template Picker (`/w/[slug]/lps/new`)

**Focal point:** the template selector and the "Continue" button.

Layout: workspace shell sidebar + main content, centered card form.

Single `<Card>` max-width 480px, `mx-auto mt-8`:
- Heading: "Generate a Landing Page" (`text-xl font-semibold`).
- Template picker: `<Select>` with all workspace templates as options — label "Template", placeholder "Select a template…". Shows template name + schema version badge in option text.
- Empty branch: if no templates exist, show `<Alert>` "No templates found. Create a template first." with a link to `/w/[slug]/templates/new`.
- LP Name input: `<Input>` type="text", label "Landing Page Name", placeholder "e.g. Grécia Jun/2026", helper "This name identifies your LP in the catalog." Required validation on submit.
- CTA: "Continue" button (primary/accent) — disabled until template selected AND name filled. Navigates to the dynamic form.

### GEN-02 / GEN-03 / GEN-04 / AST-01 / BRD-02 — Dynamic Form (`/w/[slug]/lps/new/[templateId]` or a unified `/w/[slug]/lps/generate`)

**Focal point:** the form fields. This is the core UI of Phase 4.

Layout: workspace shell sidebar + main content.

```
┌──────────────────────────────────────────────────────────────────┐
│  Sidebar (240px)     │  Main (flex-1, px-8 py-6)                │
│  - Templates         │  ┌──────────────────────────────────────┐ │

│                      │  ├──────────────────────────────────────┤ │
│                      │  │ [Field groups & repeater sections]   │ │
│                      │  │  ...                                 │ │
│                      │  ├──────────────────────────────────────┤ │
│                      │  │ [Generate LP]  (sticky footer CTA)   │ │
│                      │  └──────────────────────────────────────┘ │
└──────────────────────┴──────────────────────────────────────────┘
```

**Form max-width:** `max-w-[720px]` — centered in the content area with `mx-auto`.

#### Brand Globals display (read-only, always at top of form)

A `gray-50` panel with `border border-gray-200 rounded-md p-4 mb-6`:
- Section heading: "Brand Globals" (`text-sm font-semibold text-gray-500 uppercase tracking-wide`).
- Three rows: `brand.logo`, `brand.primary_color`, `brand.whatsapp` — each showing label + resolved value in `font-mono text-sm text-gray-600`.
- If a value is unset: show "(not configured)" in `text-gray-400 italic`.
- Read-only — no inputs. Link "Configure brand →" at bottom right (`text-sm text-gray-500 underline`).

#### Field groups (non-repeater fields)

All non-repeater, non-global fields appear in a single `<Card>` or separated by `<Separator>` elements grouped by logical proximity (planner decides grouping; default is a flat list).

Per field type:

| Field Type | Input Component | Notes |
|-----------|----------------|-------|
| text | `<Input type="text">` | Label above, helper below |
| rich text | Tiptap `<EditorContent>` with StarterKit | `min-h-[120px]`, `border border-input rounded-md p-3`, toolbar: Bold, Italic, BulletList, OrderedList, Link (5 controls) |
| image | Custom `<ImageUploadField>` (see Image Upload below) | |
| color | `<Input type="text">` + 24px live swatch (same as BrandConfigForm) | Placeholder "#0f172a", hex-only validation |
| button+URL | Two stacked inputs: `<Input>` for button label + `<Input type="url">` for URL | Label "Button Text" + "Button URL" |

Required fields: asterisk `*` in label (`text-red-500`, `aria-label` includes "required"). Validation error shown inline below the input as `text-sm text-red-600 mt-1` with `role="alert"`.

#### Repeater blocks

Each repeater block renders as a collapsible `<Card>` section:

```
┌──────────────────────────────────────────────────────────┐
│ ▼  Itinerary Days  (3 items)          [+ Add Day]        │
├──────────────────────────────────────────────────────────┤
│ ┌──────────────────────────────────────────────────────┐ │
│ │ Day 1                              [↕ drag] [× Remove]│ │
│ │  [fields for this item...]                           │ │
│ └──────────────────────────────────────────────────────┘ │
│ ┌──────────────────────────────────────────────────────┐ │
│ │ Day 2                              [↕ drag] [× Remove]│ │
│ │  [fields for this item...]                           │ │
│ └──────────────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────────────┘
```

- Repeater group header: chevron toggle (`ChevronDown`/`ChevronRight`, lucide) + repeater name (capitalized from schema field name) + item count badge (`text-sm text-gray-400`) + "Add {Label}" button (secondary/outline, `Plus` icon, lucide).
- Collapsed: shows header + item count only. Expanded by default on first render.
- Each repeater item: bordered sub-card (`border border-gray-200 rounded-md p-4 mb-2`). Header row: "{Repeater Name} {N}" label (`text-sm font-semibold text-gray-700`) + drag handle icon (lucide `GripVertical`, `text-gray-300`) + remove button (lucide `X`, `text-gray-400 hover:text-red-500`, aria-label "Remove {N}").
- Add button: appends a new blank item to the array (`useFieldArray append()`). Item count increments.
- Remove button: removes the item (`useFieldArray remove(index)`). Confirms inline (no dialog needed — no data loss since it's just form state, not persisted yet). If the last item is removed, repeater section shows "No {items} added yet. Click '+ Add {Label}' to start." in `text-sm text-gray-400 italic`.
- Drag-to-reorder: `useFieldArray move()` triggered by drag-and-drop. Visual: `GripVertical` cursor, item gets `border-blue-300 shadow-sm` while dragging. v1 implementation: drag within the list using @hello-pangea/dnd or native HTML5 drag — planner's choice.
- Minimum items: 0 (all repeaters can be empty).

#### Form submit controls

- "Generate LP" button (primary/accent) at bottom of form, full-width on mobile, right-aligned `px-8 py-2` on desktop.
- Sticky bottom bar on scroll: when the form is taller than the viewport, the "Generate LP" button becomes sticky at the bottom of the main content area (`sticky bottom-0 py-4 bg-white border-t border-gray-200`).
- On click: React Hook Form `handleSubmit` triggers Zod validation. Inline errors appear. If valid, Server Action fires.
- Pending state: button `disabled`, label "Generating…", lucide `Loader2` spinner.

- Icon: lucide `UploadCloud` 32px `text-gray-300`.

- Upload text: "Uploading… {N}%" (`text-sm text-gray-500`).
- Cancel button (text link, `text-sm text-gray-400 hover:text-gray-600`).

**Uploaded (success):**
```

│  [thumbnail 48×48]  filename.jpg             │

└──────────────────────────────────────────────┘
```
- `bg-green-50 border border-green-200 rounded-lg p-3 flex items-center gap-3`.
- 48×48px image thumbnail (`object-cover rounded`).

**Error:**
- `bg-red-50 border border-red-300 rounded-lg p-3`.
- lucide `AlertCircle` 16px `text-red-500` + error message (`text-sm text-red-600`).
- "Try again" link (text link) that re-opens the file picker.

Error messages (specific, per error type):
- Magic-bytes mismatch: "File does not appear to be a valid image. Try a different file."
- Size exceeded: "File exceeds the 5 MB limit. Compress or resize the image and try again."
- Invalid type: "Only PNG, JPG, and WEBP images are accepted."
- Upload failure (network/S3 error): "Upload failed. Check your connection and try again."

### LP-03 — Duplicate

Trigger: "Duplicate" in the LP card kebab menu on the LP list page.

Interaction: inline confirmation — no dialog. Clicking "Duplicate" immediately fires the Server Action which creates "Copy of {name}". On success: toast "Duplicate created." The new LP card appears in the grid (optimistic or via revalidate). On error: toast "Failed to duplicate. Try again."

No separate page or dialog for duplicate in v1.

### LP-04 — Export ZIP

Trigger: "Export ZIP" button in:
1. LP card kebab menu (LP list)
2. Preview toolbar

Interaction:
- On click: button enters loading state ("Exporting…", `Loader2` spinner).
- Server Action / route handler: runs render, downloads images, builds ZIP via `archiver`, streams response.

- On success: button returns to normal state, toast "Export ready — downloading." (auto-dismiss 3s).
- On error: toast "Export failed. Try again." + button re-enables.

## Interaction Contracts

### Generate LP flow (GEN-01 through GEN-04)

1. User selects template on picker page, enters LP name → clicks "Continue".
2. Dynamic form renders from template schema + metadata overlay. Brand globals displayed read-only at top.
3. User fills fields. Repeater items: add via "+ Add {Label}" button, remove via "×" icon on each item.
4. Rich text fields: Tiptap editor, 5-control toolbar. No Markdown — WYSIWYG only.
5. Image fields: drag-and-drop or click-to-upload. Presigned PUT to S3-compatible bucket.
6. User clicks "Generate LP".
7. React Hook Form `handleSubmit` runs Zod validation (required fields, URL format for button+URL type, hex format for color type).
8. If validation fails: inline errors appear, page scrolls to first error. Button stays enabled.
9. If valid: button enters "Generating…" state. Server Action fires: `render(snapshotMarkup, values, liveBrand) → HTML`, LP record created in DB (values jsonb + markup snapshot + schema_version).
10. On success: redirect to `/w/[slug]/lps/[newLpId]/preview` with toast "LP generated successfully."
11. On server error: toast "Failed to generate LP. Try again." Button re-enables.

### Image upload sub-flow (AST-01)

1. User drops file or clicks upload zone.
2. Client validates: MIME type (accept list), file size (≤ 5 MB client-side pre-check). Immediate client error on reject.
3. If client-valid: request presigned PUT URL from Server Action (server validates magic bytes + size/pixel caps server-side).
4. If server rejects: display specific error message inline.
5. If server-valid presigned URL returned: browser PUTs file directly to S3-compatible bucket. Progress bar updates via XHR `progress` event.
6. On PUT success: image URL stored in RHF field value. Thumbnail renders in success state.
7. Form submit includes image URLs as string values (already uploaded).

### Edit + Regenerate flow (LP-02)

1. User clicks "Edit" from LP list or preview toolbar → navigates to edit page.
2. Form pre-populated with saved `values` jsonb.
3. If schema version mismatch: amber alert shown, "Apply new version" action available.
4. User modifies fields, clicks "Save Changes".
5. Same validation flow as Generate.
6. Server Action: `render(snapshotMarkup, reconciledValues, liveBrand) → HTML`, LP record updated.
7. On success: redirect to preview with toast "Changes saved."
8. On error: toast "Failed to save changes. Try again."

### Delete LP flow

Trigger: "Delete landing page" in kebab menu on LP card.

Dialog (`<Dialog>`):
- Title: "Delete landing page?"
- Body: "This will permanently delete \"{name}\" and cannot be undone. Any exported ZIPs you've already downloaded are unaffected."
- Confirm: "Delete landing page" (`variant="destructive"`).
- Cancel: "Keep landing page".
- On confirm: Server Action fires. Card removed from grid (optimistic). Toast "Landing page deleted."
- On error: toast "Failed to delete. Try again." Dialog stays open.

## Accessibility

- All interactive elements have visible focus rings (`focus-visible:ring-2 ring-gray-900 ring-offset-2`).
- Required form fields: `aria-required="true"` on the input element.
- Inline validation errors: `role="alert"`, referenced by the input via `aria-describedby`.
- Image upload drop zone: `role="button"` with `aria-label="Upload image"` on the outer container; keyboard-activatable via Enter/Space.
- Upload progress bar: `<Progress aria-label="Upload progress" aria-valuenow={pct}>`.
- Repeater add button: `aria-label="Add {Label}"` (e.g. "Add Day").
- Repeater remove button: `aria-label="Remove {N}"` (e.g. "Remove Day 1").
- Preview iframe: `title="Landing page preview"`.
- Dialog: shadcn `<Dialog>` provides `role="dialog"`, `aria-labelledby`, `aria-describedby`.
- Alert: shadcn `<Alert>` provides `role="alert"`.
- Brand globals panel: `aria-label="Brand globals (read-only)"`.
- Image thumbnails: `alt="{filename}"`.
- Minimum color contrast: body text on white/gray-50 surfaces maintains WCAG AA (gray-600 `#4b5563` on white = 7.2:1).
- Color input swatch: `aria-label="Color preview"`, `role="img"`.
- Tiptap editor wrapper: `aria-label="{field label} rich text editor"`.
- Keyboard-navigable repeater drag-and-drop: native drag-and-drop is optional enhancement; move-up/move-down icon buttons (`ChevronUp`/`ChevronDown`, lucide) provided as keyboard-accessible alternative.
