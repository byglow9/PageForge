## Phase Goal

**As a** marketing team member, **I want to** generate a complete landing page by filling a dynamic form from a template, **so that** I can produce layout-faithful HTML landing pages without writing code.

<objective>
Deliver the complete form→merge→preview vertical slice: template picker, dynamic form with all field types (text, richtext, color, button+URL) and repeater blocks, brand globals display, LP generation, preview, edit, and duplicate.

Purpose: This is the core product promise. After this plan, a user can select a template, fill the generated form, see their LP rendered in a preview iframe, re-edit it, and duplicate it. Image upload and ZIP export are separate slices (Plans 03 and 04).

Output: lib/lps/actions.ts (Server Actions), six client components (LpForm, LpCard, LpPreview, RepeaterBlock, RichTextField, BrandGlobalsPanel), five RSC pages (LP list, template picker, new form, preview, edit), sidebar nav link.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client→Server Action | Form values (LP name, field values, templateId, lpId) are untrusted client input |
| Server Action→engine | markupSnapshot from DB (trusted — stored at generation time), values from form (sanitized by engine per-type) |
| RSC→Client Component | HTML string passed as srcdoc to iframe — must be server-rendered, never client-rendered |
| iframe→page | Preview iframe has sandbox="allow-same-origin" blocking script execution |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-02-01 | Spoofing | LpPreview iframe | mitigate | iframe has sandbox="allow-same-origin" attribute — scripts blocked in preview. srcdoc is set by RSC server render, never by client input directly. |
| T-04-02-02 | Tampering | generateLpAction values input | mitigate | Values pass through engine's per-type sanitization (sanitizeRichText for richtext, sanitizeUrl for image/button, sanitizeCssColor for color) during renderLp(). Engine uses LiquidJS outputEscape for text. |
| T-04-02-03 | Tampering | Tiptap rich text HTML output | mitigate | RichTextField outputs HTML via editor.getHTML(); this value is stored in LP.values and passed to engine on generate. Engine's sanitizeRichText (sanitize-html with allowlist) is applied server-side during render, not client-side. |
| T-04-02-04 | Elevation of Privilege | lpId / templateId from URL params | mitigate | All DB queries use withTenantDb which injects workspaceId from server session; findById always includes workspaceId in the where clause — IDOR prevented at app level with RLS backstop. |
| T-04-02-05 | Elevation of Privilege | LP list / get access | mitigate | requireWorkspace for reads (all members); requireWorkspaceRole(["owner","admin","editor"]) for mutations. Can function gates conditional rendering of create/edit/delete UI. |
| T-04-02-06 | Tampering | Schema version mismatch (D-08) reconciliation | mitigate | reconcileLpValues runs server-side in updateLpAction; old keys not in the new schema are dropped, preventing stale values from polluting the stored JSON or the render scope. |
