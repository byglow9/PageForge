<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser→Server Action | filename, contentType, fileSize, firstBytes from client — all untrusted |
| Server Action→S3 | presigned URL uses SigV4; expires in 1 hour; Content-Type is signableHeader |
| browser→S3 bucket | Direct PUT with presigned URL; S3 enforces Content-Type via signed headers |
| S3 URL→engine | publicUrl stored in LP.values; passed to sanitizeUrl() during render |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-03-01 | Tampering | requestPresignedUploadAction — malicious file disguised as image | mitigate | Server-side magic-bytes validation via file-type fileTypeFromBuffer(). Client MIME type string is untrusted and checked for UX only; server re-validates independently. |
| T-04-03-02 | Tampering | Presigned PUT URL abuse (another tenant using the URL) | mitigate | Presigned URL is scoped to a specific S3 key under workspaces/{workspaceId}/. An intercepted URL can only PUT to that exact key in that workspace's prefix. URL expires in 3600 seconds. |
| T-04-03-03 | Information Disclosure | S3 bucket CORS misconfiguration | accept | CORS configuration on MinIO/S3 bucket must allow PUT from the app's domain. Documented in .env.example; production bucket policy is a deployment concern outside Phase 4 scope. |
| T-04-03-04 | Tampering | Image URL in LP values injected into HTML | mitigate | Engine passes image field values through sanitizeUrl() which blocks javascript: and data: URIs; outputEscape provides additional HTML entity escaping. publicUrl is an S3 HTTPS URL that passes sanitizeUrl. |
| T-04-03-05 | Denial of Service | Large file upload exhausting memory in requestPresignedUploadAction | mitigate | Server only receives first 4100 bytes (magic-bytes buffer) plus metadata; actual file bytes go directly to S3 via presigned PUT — app server memory is unaffected by file size. |
| T-04-03-06 | Elevation of Privilege | Accessing other workspace's S3 objects via path traversal | mitigate | S3 key is constructed server-side using ctx.workspaceId from requireWorkspaceRole — never from client input. crypto.randomUUID() generates the filename component. Path components validated implicitly by UUID format. |
</threat_model>
