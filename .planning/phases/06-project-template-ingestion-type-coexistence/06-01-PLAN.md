| Boundary | Description |
|----------|-------------|
| DB → render path | lp.kind from DB reaches renderLp(); a corrupt DB value must not bypass the guard |
| Template list → UI | kind from listTemplatesAction reaches TemplateCard; no user input can influence this in this plan (pure additive column) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Tampering | renderLp() type boundary guard | mitigate | Guard throws explicitly before any LiquidJS processing — cannot be bypassed by wrong kind value from DB; covered by boundary test in Plan 02 |
| T-06-02 | Tampering | ALTER TABLE ADD COLUMN migration | mitigate | TEXT + CHECK constraint enforces 'LIQUID' and 'VITE_SPA' at DB level; any other value rejected by Postgres before reaching application layer |
| T-06-03 | Information Disclosure | kind column in catalog | accept | kind is not sensitive metadata; it identifies the template type to workspace members who already have read access |
| T-06-04 | Elevation of Privilege | TenantTemplateHelpers.create kind parameter | mitigate | kind is passed from server-session-derived action (Plan 02), never from client payload directly; this plan only adds the optional field to the type signature |
</threat_model>
