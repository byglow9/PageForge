## Phase Goal

**As a** workspace member, **I want to** upload a pre-built Vite `dist/` ZIP as a project template, **so that** I can register a Lovable-exported project in the PageForge catalog alongside LIQUID templates — with security scanning and tenant isolation from day one.

<objective>
This plan delivers the VITE_SPA ingestion vertical slice: a user visits `/w/[slug]/project-templates/new`, fills in a name, selects a ZIP file, and submits. The server validates the ZIP (zip-slip, zip-bomb, index.html presence), scans for embedded credentials (Supabase JWT, Stripe live key, AWS access key, Lovable URLs), uploads every dist/ file to S3 under a tenant-scoped non-enumerable prefix, creates a Template row with kind='VITE_SPA', and returns any scan findings as warnings. The UI surfaces warnings as a toast before redirecting. A dedicated Vitest test asserts the type-boundary guard added in Plan 01 (V2-11): renderLp() throws on VITE_SPA input and passes on LIQUID input.

Purpose: Closes PRJ-02 (validation + scan + storage), the ingestion side of PRJ-01 (VITE_SPA template creation from dist/), and V2-11 (type-boundary test). Plan 01 must be complete (kind column migrated, Prisma client regenerated, renderLp() guard in place) before this plan runs — the template.create call passes id: templateId and kind='VITE_SPA' which requires the extended TenantTemplateHelpers.create signature and the regenerated Prisma client.

</objective>
